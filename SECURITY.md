# Security Policy

## Reporting

If you find a security issue in repository content, templates, or published installation instructions, report it privately to the maintainers if a private channel exists.

If no private channel is available:

- avoid posting secrets or exploit details publicly
- open a minimal public issue that asks maintainers for a secure contact path

## Scope

This repository mainly contains skills, templates, and documentation. Security issues may include:

- unsafe installation instructions
- secrets accidentally committed to templates or examples
- generated code patterns that encourage obvious insecure defaults
- supply-chain risks in recommended dependencies

## Response Expectations

Maintainers should acknowledge reports, assess scope, and update templates or docs quickly when the issue affects the recommended golden path.
