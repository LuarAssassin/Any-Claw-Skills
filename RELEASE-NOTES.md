# Release Notes

## v0.1.0 - Claude Code First Public Release

### Highlights

- Reframed the repository as a Claude Code first skill package rather than a generic template dump
- Defined a release-verified golden path: `Standard + Python + OpenAI + CLI + Telegram + Productivity`
- Added a published support model with `GA`, `Beta`, and `Preview` tiers
- Added release-facing docs for support policy, release checks, testing, and domain pack contracts
- Tightened the project around reproducible assistant builds driven by skills, commands, and templates

### What v0.1.0 Promises

- Claude Code users get the primary supported installation and usage path
- Maintainers get a release checklist and structured verification flow
- The repository clearly states which parts are stable, which are partial, and which are reference-only

### What v0.1.0 Does Not Promise

- Official marketplace publication has been completed
- Every client platform has equal release maturity
- Every stack, channel, provider, and domain combination is equally validated

### Upgrade Notes

- Treat Claude Code as the release target for this version
- Prefer the golden path for demos, screenshots, examples, and verification
- Use the support matrix before expanding GA claims
